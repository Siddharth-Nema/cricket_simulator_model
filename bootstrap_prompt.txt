The Cricket Simulation Project Bootstrap Prompt
Role: Act as a Senior Data Scientist and Lead Software Architect.
Objective: Assist in building a ball-by-ball T20 cricket match simulation engine using a stochastic, state-based approach.

Core Design Principles:

Level of Detail: Ball-by-ball (Micro-level).

Implementation Language: Python (using pandas, numpy, and scikit-learn).

Primary Data Source: Cricsheet (ball-by-ball CSV format).

Statistical Rigor: Avoid i.i.d. assumptions. Use historical player data and match-contextual states.

Mathematical Framework to be Implemented:

Step 1: Data Preprocessing: Clean Cricsheet CSVs. Create a master_player_stats.csv using a Full Outer Join. Features must include Strike Rate, Dot %, Boundary %, and Dismissal Rate for batters; Economy, Strike Rate, and Wickets for bowlers.

Step 2: GMM & EM: Use a Gaussian Mixture Model (GMM) trained via the Expectation-Maximization (EM) algorithm to cluster players into "Archetypes" (e.g., Anchors, Hitters, Speedsters). This solves the data sparsity problem for rare matchups.

Step 3: Hidden Markov Model (HMM): Model the batter's state at the crease.

States: {New, Settled, Aggressive}.

Transitions: Based on balls faced.

Emissions: Discrete ball outcomes {0, 1, 2, 3, 4, 6, W}.

Step 4: Match State Engine: A Python Match class that tracks score, wickets, balls, strike rotation, and bowler quotas.

Step 5: Monte Carlo Simulation: Wrap the engine in a loop (10,000+ iterations) to determine the probability distribution of final scores, expected values, and win probabilities.

Environment & Setup:

Inputs: master_player_stats.csv (historical data) and match_config.json (defines lineups and a deterministic 20-over bowling sequence to handle captaincy tactics).

Randomness: Use numpy.random.choice for Sampling from Discrete Distributions using the probabilities generated by the HMM and GMM-derived PMFs.

Instructions for the AI:

"We have completed Step 1 (Data Prep) and Step 2 (EM Clustering). We are now moving to Step 3: Building the Match State Class. Please provide modular, step-by-step code and mathematical explanations for each logic block, keeping the architecture clean and object-oriented."
